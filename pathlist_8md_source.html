1 Overview of path-returning methods
2 ==================================
3 
4 This section lists Pathie’s methods for retrieval of system or user
5 standard pathes along with possible examples. It might serve as an
6 overview if you don’t want to read the documentation of all those
7 methods.
8 
9 The _Standard_ column denotes the standard the specified path was taken from under UNIX; Windows
10 has no applicable standards it follows for its directory structure. The _Type_ column
11 specified whether this path varies on a per-user basis (_User_) or is a fixed system
12 path that is the same for all users (_System_); _Other_ means the path is unrelated
13 to User information.
14 
15 On Windows, several pathes may share the same location. This is because Windows
16 doesn’t have such a detailed filesystem layout structure as UNIX systems have.
17 If your application needs to run on Windows also, you should be careful to avoid
18 file naming conflicts in those directories.
19 
20 The global pathes given below for UNIX are those you’ll receive if you called Path::set_global_dir_default()
21 with `LOCALPATH_NORMAL` as its argument. You can also use `LOCALPATH_LOCAL` to instruct
22 the `global_*_dir()` methods to return the corresponding pathes under the system-local
23 hierarchy (mostly `/usr/local` and `/var/local`) instead.
24
